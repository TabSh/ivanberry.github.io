清除浮动

在页面布局时，存在多栏布局的情况下，我们大多会采用float的技术，使得页面完美呈现我们所需要的状态，同时，问题也伴随而来，这样的问题就引入了一个前端开发中经常用的技术：浮动清楚

伴随的问题 

	
	<div class="container">
	<div class="sideBar">Lorem ipsum dolo</div>
	<div class="main">MAIN</div>
	</div>
		
	.sideBar, main {
		float: left;
	}

具体表现问题如上图，当未定义容器高度时，或者说容器高度不能容下子元素时，溢出产生了。这就需要专门的技术来处理，一般称之为“浮动清楚”，下面我们全面介绍一下各种清楚技术，并分析优缺点，得到最适合的方法。

常见浮动清楚一览

  • 空块添加
  • 父容器浮动
  • overflow属性
  • 伪元素添加
  • 本质解决方法

上述5种方法，基本揽括了常见的浮动清除办法，由一定的参考价值，我们一一来解读。

空元素块添加

	<div class="clearfix"></div>

	.clearfix {
		clear: both;
	}

很明显，这个方法很有效，原理也很简单，空元素块的clear属性添加使它占满父容器宽度，使得其左右都不能存在任何其他元素。方法有效的同时，同样也有它的问题，为了清楚浮动而带来的多余代码是令人困扰的，如果有更好的方法，在有效清除浮动的前提下语义化标签，放弃过多语义不清的代码是科学的。

优缺点：

  • 优点：简单有效
  • 缺点：增加语义不明显代码，文档冗余

父容器浮动

这种方法其实是最不适合的，本身float技术就是出于滥用的阶段，使用这种方法更是饮鸩止渴，不值得提倡，不考虑这些的前提下，它也是一种有用的方法。


	.container {
		float: left;
		width: 100%;
			}

overflow属性添加

方法很有效，但是偏离了overflow属性的应用，可能发生子元素被切割的情况，也是酌情使用吧。 


	.container {
		width: 100%;
		overflow: hidden;
		display: inline-block; //for IE 
		display: block; //elements are displayed as block.
	}

	//另外一种适应于IE
	.container {
		width: 100%;
		overflow: hidden;
		_overflow: visible; // for IE
		_zoom: 1; //for IE 
	}

伪元素添加方法

这种方法可能少见，就目前而言，个人觉得是最好的，没有多余的语义不明的标签，也没有看起来怪怪的css语法，对现代浏览器而言，适用性都很强，浏览器市场中绝大多数都很好使用这种方法来清除浮动，值得学习并应用于项目中。 


.container:after {
	content: "";
	display: table;
	clear: both;
	zoom: 1; //for IE	}

这个方法是衍生于一种称为"Micro Clearfix"的浮动清楚方法，它由Niclas Gallagher创造出来的,其实这种方法原理和html文档中添加一个空的元素块是一样的，只是得意于伪元素的存在，我们不需要在html文档中额外添加语义不明的标签，虽然在css中相比而言多了几条语句，但是总是值得的。 


.container:before, .container:after {
	content: "";
	display: table;
	}
.container:after {
	clear: both;
	}
.container {
	zoom: 1; //for IE
	}

本质解决办法

何为本质、彻底的解决办法呢？上面介绍的各种方法，都是为了清除浮动，而清除浮动是因为我们采用了float来对页面布局，那答案就自然流露出来了，只要是不采用这种方法，并且还需要实现相当的布局，是不是就能彻底放弃更新、学习各种奇奇怪怪的清楚浮动的方法呢？用最简单的办法达到目的，而且效果还不错。下面我们来试试用flexbox布局。对于flexbox的基础知识，可以看我的另外一篇文章： flexbox学习小指南。 

	
.container {
	display: flex;
}

上面的代码及其的短，但是已经解决我们的问题啦，flex就能很好的实现多栏布局，下面我会给出稍微详细的细节，更多的了解需要知道flex为何物，不然理解起来会怪怪的，更多详情可以参考W3C和MDN。 

	
.container {
	border: 1px dashed red;
	padding: 10px;
	display: flex;
}

.sideBar {
	border: 1px solid green;
}

.main {
	border: 1px solid blue;
}

上面的介绍简单了，实现多栏布局，可以使用float的布局，但是使用float布局会带来很多的问题，至少需要额外的代码来解决浮动的清除，而在前端迅速的发展下，随着浏览器的快速更新，W3C标准的推行，新的方法会更好的、更有效的替代过去一些当下不那么适宜的方法，例如采用flex布局，我们至少可以不用担心浮动溢出的问题，更为重要的是，在详细了解flex布局后，你一定会说：我了个去，这么厉害。新技术的使用有一个最大的问题就是兼容性，其实flex很早就已经提出了，当下主流浏览器对它真是没有什么问题，大家可以看看CANIUSE的统计结果。

结论

经过上面的介绍，个人对浮动的清楚或者是多栏的布局有了很深刻的理解，我就个人使用的优先级做个结论,当然不会每个都使用，有些不值当的方法可以直接淘汰，比如同样也浮动父容器。

  1. flex布局
  2. 伪元素添加
  3. overflow属性
  4. 空元素块添加
  5. 父容器浮动
