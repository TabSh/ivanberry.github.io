浮动

浮动与定位在CSS中属于基础，但是又是很重要的一部分，对于布局定位有很重要的意思，必须要搞清楚，必须。 对于float 和 position的中文译名我不是特别熟悉（又开始装逼了），就暂定为浮动和定位吧。

先来个引用，客官且看

A CSS selector is a pattern rule that matches style rules to a specific element (or elements). The selector begins a rule set and precedes definitions of how the element(s) should look.
W3C

对于浮动的值，我们就不列出来了，很多地方都有非常详细的资料，咱们不做多余事情，浮动，首先要有一点关键的认识：浮动的元素一定程度上脱离了普通 文本流（一定程度？）。这样带来的效果就是，它本身脱离了，但是却还是影响普通文本流其他元素的布局。而另外一点是：浮动元素的外边距不会聚合。

浮动有了，但是有时候会出现一下这种情况，但被浮动元素存在height或元素本身被内容撑开时，父级元素不能随子元素高度变化，这也是 浮动元素脱离了普通文本流的一个证据。当然，这种情况可能不是我们所需，就需要我们浮动清除上场了。浮动清除的方法不少，下面一一简单介绍。

  • 父级元素浮动
  • 父级元素添加overflow属性
  • 添加浮动清除专属块

父级元素浮动

这个好理解，我的子元素浮动了，不完全归我管了，那我也浮动，你们小子就跑不了了的。浮动会增加性能上的负担， 不必要的地方或者有更好方法解决时候不推荐使用这种清除方法。

See the Pen fix float-2 by Tab (@ivanberry) on CodePen.

添加浮动清除专属块

这个方法也很简单，在浮动元素同级别添加一个专属用于清除的块，为了更广泛的使用，可以给来个 class，这样方法很好用，但是有个一个缺点，语义不明显，且添加多余的HTML语句，但是不能因噎废食 ，一个简单的块添加就能完美的解决浮动带来的问题。

See the Pen clear float-1 by Tab (@ivanberry) on CodePen.

父级元素添加overflow属性

这个方法是在浮动对象的父级元素中添加overflow: hidden;，本身overflow是用于解决子元素溢出问题的 却也可以用来清除浮动，这个方法没有添加额外的HTML语句，推荐使用。
