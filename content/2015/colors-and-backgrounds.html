颜色和背景属性

颜色

element {
    color: keywords | grb | grba | hsl | hsla;
}

color属性主要适用于前景色设置，如文字颜色，边框颜色等。

color颜色本身没有太多需要讲解的，它多用于文字前景颜色的设定，还有超链接多种状态颜色的设定，还有使用于border颜色的设定。

背景

背景是指包括context-box和padding的全部，这是背景的范围定义。背景有几个可设定属性：

  • background-color
  • background-image
  • background-repeat
  • background-attachment
  • background

background-color顾名思义就是对背景颜色的定义，注意背景的区域性

background-color:

element: <color> | transparent | inherit;
initial value: transparent
applies to: all elements
inherited: no

颜色本身就是很大的学问，可以参照维基百科：Web_colors

关于颜色的不可继承性需要说一点，初学的我一开始也有误会，稍微提一下，当我们设定body为某颜色时，一下的元素在没有设置background-color的前提下会“继承”body的颜色，这跟我们提到的background-color属性有出入啊！其实，我们应该关注到一点背景颜色的默认值, 它是透明,这样我们就能想象得到，background-color本身确实是不能继承的，某元素呈现出了父级元素的背景色，是因为它本身背景色是透明的。

background-image:

values: <uri> | none | inhert
initial value: none
Applies to: all element
inherited: no

See the Pen rOgjGq by Tab (@ivanberry) on CodePen.

因为默认值的存在，我们若是需要某元素有背景图像就必须声明，建议在声明背景图像的同时使用背景颜色，作为背景颜色加载失败时的备选。强调一点，背景图像的呈现总是在背景颜色之上，对于GIF、JPEG等，背景图像会覆盖背景颜色，而对于PNG存在透明的，背景色仍然会显示出，这个颜色的继承问题有类似。

关于背景图像位置的属性，有关键字（left,center,right etc),也有距离值，例如10px,1em等等，还有百分值，每一种有细微差别。

强调一下：背景图是存在于包括padding。也就是说，它的初始位置0%，0%时，对元素本身而言，是包含有padding部分的。不同值有细微不同，关键字值可以成对或者不成对出现，而且它们可以以任何顺序出现。数值对：10px 20px ，它就是指原图本身左上角较元素（包含padding）区域左上角偏移10px,20px。最为特别的是百分值时，百分值同时作用与元素和原图本身，假如有background-position: 50%,50%，那么它有几点特别：

  • 顺序：水平先行
  • 原图本身参考点为50%,50%（图片中心）
  • 元素（包含padding)参考点为中心
  • 对其两个参考点

当使用百分数或数值时，可以使用负数使得图像移出padding，有被padding切割的效果。

background-attachment

values: scroll | fixed | inherit
initial values: scroll
applies to: all elements
inherited: no

这个属性值很好理解，不做多说。
