Flexbox学习小指南

来源： CSS-tricks
The flex layout model, the children of a flex container can be laid out in any direction, and can “flex” their sizes, either growing to fill unused space or shrinking to avoid overflowing the parent. Both horizontal and vertical alignment of the children can be easily manipulated. Nesting of these boxes (horizontal inside vertical, or vertical inside horizontal) can be used to build layouts in two dimensions.
W3C

和大家一起学习下Flexbox布局，W3C文档中有提到在CSS2.1中有四种布局模型：

  • block layout
  • inline layout
  • table layout
  • position layout

上面四种模型，我们这里不详细介绍，现在有新添了一个新的，我们接下来介绍：flex layout.个人理解flex在布局上能使以往我们很是为难 情况得以改善，而且实现也很容易，现实中Bootstrap也在一定程度上能帮到我们，但是flexout的实现非常简单。首先明确最重要一点,父级容器 添加display: flex;后，就改变它元素的行为，在容器有了一个横纵坐标系的参照，接下来，我会参照W3C标准文档，MDN文档和 天才CSS专家Chris Coyier网站学习，它们的资料对问题的表述非常到位，所以我们直接借鉴啦！

如上图，在容器添加flex属性后，容器参考系发生了变化，有了两根垂直相交的轴线，主轴和相交轴。正式学习之前就这个二维的坐标 必须说一下，以免后面学习的误会，自己本身现在还有误会，我们一起深入学习。

主轴，主轴空间
容器的主轴是元素排列的主要轴线，主轴线上的容器空间成为主轴空间。
主轴线尺寸,主轴线属性
规定子元素在主轴上所占的维度，是它的主要尺寸，根据主轴的不同，可以是子元素宽度或者高度。
主轴起点，主轴终点
轴线上的方向为从起点到终点扩展子元素

本来想把交叉轴同样列出来，但是这个时候我想到了做数学证明题时：同理可得。理解还这个最基本点后，我们再开始说道说道，基础知识

基础知识

父级属性（容器属性）

display

这个属性值，添加在父级元素上，作用很明确，这也是flex布局的开始，而效果表现在它的直接子元素。

.container {
         display: flex; /*or inline-flex*/
        }

关键属性，声明对象是容器元素，对容器元素本身没有任何影响，作用对象是它的直接子元素。

flex-direction

这个个属性定义了flex容器下子元素布局的主轴参考线，它是容器的属性，表明子元素或水平或垂直轴线上排列。

.container {
	flex-direction: row | row-reverse | column
	| column-reverse;
	}
  • row:默认值，定义主轴为水平向，自左向右（ltr)。
  • row-reverse: 对本身为自左向右书写的文字，改变为相反的排列方向。
  • column: 更改主轴为竖直方向。
  • column-reverse: 如本身是自上而下，该为相反方向。

这个属性值改变容器内子元素的排列方向，我们下面会集中举例，便于学习

flex-wrap

.container {
	flex-wrap: nowrap | wrap | wrap-reverse;
	}
  • nowrap:默认值，单行显示
  • wrap: 多行显示,默认左至右
  • wrap-reverse: 多行显示,默认右至左

容器内子元素在轴线上排列是否容许多行的出现，默认是单行显示，不允许断裂。充分利用行内的空间，大小随行内 空间的减少（增大）而减少（增大）。

flex-flow

flex-flow是flex-direction flex-wrap的缩写，定义了容器内的主轴线，默认值为row nowrap。

.container {
	flex-flow: <'flex-direction'> | <'flex-wrap'>;
	}

justify-content

定义主轴线上子元素的对齐，它表现为对轴线上未充分利用的利用，说起来好像很绕口，但是试试例子就能明白它

.container {
	justify-content: flex-start | flex-end
	          space-between | space-around;
	}
  • flex-stat: 默认值，在主轴线上从轴线起点开始向后排列
  • flex-end: 在主轴线上从轴线终点向前排列
  • space-between: 从轴线起点开始，子元素间均化利用未能利用的空白,第一个元素在轴线起点处，最后一个在轴线终点处，
  • space-around: 子元素前后（四周）均化利用未能利用的空白

align-items

当前主轴线的交叉线方向上的排列方式，属性值跟justify-content类似，从上图我们能大致了解到它的表现。

.container {
	align-items: flex-start | flex-end|
	          center | baseline | stretch
	}
  • flex-start: 主轴交叉线起点为起点排列
  • flex-end: 主轴交叉线终点为起点排列
  • center: 交叉线上居中
  • baseline: 子元素基线对齐
  • strecth: 拉伸利用容器空间，默认值

align-content

这个属性值是容器属性中相对难理解一点的，容易很justify-content混淆，其实说明白了它很容易（至少我现在是这么觉得的）， 我们和justify-content比较学习，justify-content强调或者说作用的对象是子元素本身之间的关系，是怎么利用轴线上的未利用 空间，而align-content主要是以轴线为单位，不考虑轴线上的元素，它需要和flex-flow: wrap配合使用,只有出现多行排列时， 这个属性值才有存在的意义，我们会在之后的具体例子中体会到。

.container {
				align-content: flex-start | flex-end |
				               center | space-between |
				               space-between | stretch
				}
  • flex-start:lines 从容器开始端（为什么是端，而不在是点呢？）开始排列。
  • flex-end: lines从容器底端开始排列
  • center: lines居中排列
  • space-between: lines排列间距均话，第一行排容器开始端，末行排底端
  • space-around: lines四周都是间距
  • stretch: 拉伸占满

元素属性

order

用于定义子元素页面轴线上出现的顺序，默认是按照HTML文档顺序出现，我们可以通过修改子元素本身的order属性， 在不改变源文件结构的前提下，改变某子元素出现的顺序，这个挺厉害呀！

				.item {
			order: <integer>;
				     }

flex-grow

此属性主要用于更改某子元素的大小，属性值为无单位数值，数值以比例的形式表达，有点权重的感觉，不过是空间的权重，提一点， 它不接受负值。

.item {
				flex-grow: <number>
				}

flex-shrink

定义子元素是否能被拉伸,负值无效

.item {
				flex-shrink: <number>
				}

flex-basis

这个属性值值得玩味，它定义了在重分配未利用空间前子元素默认大小，它可以是长度单位或者关键词（auto, content),当为关键词时， 默认大小就需要看子元素本身的宽，高属性,content好理解，根据内容控制。

flex-basis: {
	<length> | auto /*default*/ | content;
	}

当它为0时，不考虑多余的未利用空间（这还是一个疑惑，我等会要试试,当设置为auto时，根据 flex-grow来分配空间。

flex

flex是上面上个子元素属性值的简写，W3C推荐使用简写，简写好像很厉害的样子，可以仔细研究研究的。

.item {
	flex: none | [ <'flex-grow'> <'flex-shrink'> <'flex-basis'>];
	}

align-self

这个值可以覆盖align-items,可以完全自主的定义某一个特定的子元素，也是很好玩的一个属性。关于flex布局 的相关知识介绍的超不多了，再提一句，float,clear,vertical-algin对flex-items没 有作用啦！

我们之前提到了关于子元素一个非常好用的属性，就是在不改变HTML结构的同时，我们让子元素出现不同的顺序，应该蛮好玩的。

经过仔细翻阅W3C文档，并结合谷歌，对子元素的flex属性有点新的体会，这里更新一下，以免忘记

  • W3C推荐使用简写flex代替flex-grow,flex-shrink, flex-basis.
  • 跟进上一条，flex默认值为0,1,auto，这和单独的默认值不同，单独是对应为1,1,auto.
  • flex-grow决定元素的主轴维度的增长能力.
  • flex-shrink决定元素的主轴维度的收缩能力.
  • flex-basis决定元素的主轴维度的大小,这个大小是指为重新分配容器未利用空间前的初始大小.
    • width: 和width属性值相同
    • auto:由主轴尺寸值计算得之，若这个值本身也是auto，那么元素大小基于元素内容自动生成.
    • 赋值为0时必须加上长度单位.
  • 当前两者赋值为0时，意味着说：我不要放大也不能收缩，由flex-basis决定我们的大小.
  • 当给它们不同的值时，表明他们放大缩小的速度不同，值越大越块，增大的尺寸与flex-basis相关.

常用flex值

  • initial: 等同于flex: 0 1 auto;,这个值就表明作用元素在主轴方向不会放大以占满未利用空间，但当空间不足时会缩小.
  • auto:等同于flex: 1 1 auto;基于元素的尺寸，可缩可放，自动分配主轴方向所有未利用空间.
  • none:等同于flex: 0 0 auto;同样基于元素尺寸，完全不能缩放.
  • <positive-number>，等同于flex: <positive-number> 1 0%;,按确定比列（占所有flex-grow的比重）分配主轴未利用空间.
